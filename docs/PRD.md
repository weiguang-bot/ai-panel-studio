# AI Panel Studio — 产品需求文档 (PRD)

## 1. 产品定位与核心价值

### 1.1 产品概述

AI Panel Studio 是一个基于 AI 的圆桌讨论模拟平台。用户输入一个话题和期望的专家人数，系统自动生成由虚拟专家组成的讨论阵容，并实时呈现一场多轮次的深度讨论。讨论结束后，系统自动总结共识要点和分歧观点。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| **即时** | 输入话题后 5 秒内开始讨论，无需预约或等待 |
| **多元** | 每次生成不同立场、不同背景的虚拟专家，确保多角度视角 |
| **深度** | 3 轮结构化的讨论流程，覆盖开场、深入、总结全阶段 |
| **智能** | 基于 DeepSeek V4 Pro 模型动态生成内容，非预设模板 |

### 1.3 产品目标

- **MVP 目标**：验证 AI 实时讨论的可行性，提供可演示的最小功能闭环
- **长期愿景**：成为知识工作者获取多元观点、快速理解复杂话题的标配工具

## 2. 目标用户画像

| 用户类型 | 典型场景 | 核心需求 | 使用频率 |
|----------|----------|----------|----------|

| **技术爱好者** | 了解新兴技术的社会影响 | 轻松、有趣的知识探索 | 每周 1-2 次 |

## 3. 用户痛点分析

| 痛点 | 描述 | 解决方案 |
|------|------|----------|
| **信息茧房** | 算法推荐导致用户只看到单一视角的观点 | AI 生成多元立场的专家阵容，强制呈现多角度 |
| **决策低效** | 面对复杂话题需要查阅大量资料才能形成全面认知 | 实时讨论在 30 秒内呈现结构化观点 |
| **讨论组织成本高** | 组织一场圆桌讨论需要协调多人时间，门槛极高 | 一键创建，无需协调任何人 |
| **观点质量不稳定** | 随意讨论容易偏离主题，缺乏深度 | 3 轮结构化流程 + 基于专家立场的针对性发言 |

## 4. 核心功能列表

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 话题创建 | P0 | 用户输入话题文本 + 选择专家人数（2-10 人） |
| 专家生成 | P0 | AI 根据话题生成虚拟专家（姓名、头衔、立场、配色） |
| 实时发言流 | P0 | SSE 协议推送每秒更新的发言，支持自动滚动 |
| 多轮讨论 | P0 | 3 轮结构化讨论（开场→深入→总结） |
| 共识分歧总结 | P1 | 讨论结束后 AI 分析发言内容，生成共识和分歧清单 |
| 嘉宾席展示 | P1 | 左侧面板展示所有专家，当前发言人高亮 + 声波动画 |
| 发言活跃度 | P2 | 右侧面板统计每位专家的发言次数比例 |
| 历史记录查看 | P2 | 通过 transcript 接口获取完整发言记录 |
| 结论持久化 | P2 | 结论生成后持久化到数据库，支持幂等调用 |
| 响应式布局 | P2 | 超宽屏/桌面/平板/手机四档适配 |

## 5. 非功能性需求

### 5.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 创建会话响应 | ≤ 3 秒（含专家生成） |
| SSE 首屏延迟 | ≤ 2 秒（连接建立到第一条发言） |
| 发言间隔 | 0.1-0.3 秒（MVP 阶段，可配置） |
| 发言流吞吐 | ≥ 3 条/秒 |
| 并发连接 | ≥ 50 条 SSE（开发服务器） |
| 数据库查询 | ≤ 100ms（含分页、筛选） |

### 5.2 安全性

- API 鉴权：MVP 阶段通过 `DEEPSEEK_API_KEY` 环境变量管理
- 输入校验：所有用户输入在服务端进行长度和格式校验（topic ≤ 500 字符）
- XSS 防护：前端使用 Vue 模板自动转义
- CORS：后端 `flask-cors` 配置跨域白名单
- `.env` 文件已加入 `.gitignore`，防止密钥泄露

### 5.3 兼容性

- 浏览器：Chrome 90+, Firefox 90+, Safari 15+, Edge 90+
- Node.js ≥ 18（前端开发）
- Python ≥ 3.11（后端运行）

### 5.4 可维护性

- 代码结构：蓝图模式 + 服务层分层
- 数据库：SQLite，文件级，零运维
- 测试覆盖：55 个 pytest 测试覆盖核心 API
- 日志：Flask 开发模式自带请求日志
